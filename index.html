What is shrift?

shrift is a minimal spec framework for testing your infrastructure based on bash.
shrift installs as a single script and executes bash one-liners as a test suite.
No complex setup. No DSL.

# example spec file
test -f /etc/passwd
whoami | grep root
/usr/local/bin/my_test_script
netstat -lntp | grep 80

$ ./shrift
....
4 tests, 0 failed

Quickstart

Download shrift.

# download the script
curl -L -o shrift https://github.com/thisendout/shrift/raw/master/shrift
# make executable
chmod +x shrift

Write and run your first spec.

# write a passing test
echo "test -x ./shrift" > test_spec.sh
# execute with shrift
./shrift test_spec.sh

Documentation

For more resources on using and running shrift, please consult the project GitHub page.

Spec Files

Files to be executed by shrift must contain one test per line. A test is considered passing when the return code is 0. Any other condition is considered a failure.

# example spec file
test -f /etc/passwd
whoami | grep root
/usr/local/bin/my_test_script
netstat -lntp | grep 80
